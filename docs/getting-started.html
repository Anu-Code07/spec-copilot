Getting started

Install SpecDrive and run your first frontend spec workflow.

Prerequisites

  • Node.js 20+
  • An existing Flutter, Next.js, or React Native project
  • For CLI AI generation: GEMINI_API_KEY (free tier) or Ollama

1. Install

# From npm (after publish)
npm install -g @specdrive/cli

# Or from source
git clone https://github.com/Anu-Code07/spec-copilot.git
cd spec-copilot && pnpm install && pnpm build

2. Initialize your project

cd your-flutter-app
spec init --stack flutter   # or nextjs | react-native

Creates .specdrive/ with steering files, config, and empty specs/ folder.

3. Configure LLM (CLI mode)

Add to ~/.zshrc (macOS/Linux) or PowerShell profile (Windows):

export GEMINI_API_KEY="your-key"   # https://aistudio.google.com
MCP mode

If you use Cursor or Claude via MCP, you do not need a SpecDrive LLM key — your host AI generates the documents.

4. Create a spec

Quick mode (all docs at once)

spec create "Product Review Screen" --quick
spec implement --spec product-review-screen --next

Gated Kiro-style workflow

spec create "Product Review Screen"
spec approve requirements --spec product-review-screen
spec gap-analysis --spec product-review-screen
spec approve gap-analysis --spec product-review-screen
spec design --spec product-review-screen
spec approve design --spec product-review-screen
spec tasks --spec product-review-screen
spec approve tasks --spec product-review-screen
spec implement --spec product-review-screen --next

5. Validate setup

spec doctor
spec list
spec status --spec product-review-screen
